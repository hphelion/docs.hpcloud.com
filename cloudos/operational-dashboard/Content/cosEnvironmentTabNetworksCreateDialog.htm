Create Network Dialog

To access: Select the Environment tab > Networks > click Create Network.

Caution: The Create Network button only displays while the networks are editable. Once the install process is triggered by the Complete Install (see Complete Install Overview), this button disappears and no changes can be made. If you want to make changes, you need to start over by re-installing the HP Cloud OS Administration node. See the HP Cloud OS Documentation web site for instructions.

The following table describes the Create Network dialog UI elements:

UI Elements Description
Network Tab
Type

Enter the network type.
Logical Interface

Select one of these values associated with the network type:

  • bmc
  • intf0
  • intf1
  • intf2
Subnet IP address associated with the network type.
Netmask Enter the 32-bit number used to distinguish the network and host portions of an IP address.
Bridge Enabled

Select True or False.

  • False Recommended -- Do not add a bridge interface on top of the interface for the corresponding network.

  • True -- Add a bridge interface on top of the interface for the corresponding network.
VLAN Tab
VLAN ID Enter the Virtual Local Area Network ID number associated with the network type.
VLAN Enabled

Select True or False.

  • False Recommended -- The associated network exists on a separate physical interface from the other networks.

    If the networks (particularly os_sdn and public) exist on separate physical interfaces,VLAN Enabled must be set to False. This means that the Network Mode will be set to dual by default on the Confirm Complete Install dialog.

  • True --The associated network coexists on the same physical interface as other networks.

    If the networks (particularly os_sdn and public) co-exist on the same physical network, VLAN Enabled must be set to True. This means that the Network Mode will be set to single by default on the Confirm Complete Install dialog.

Router Tab
Router Enter the device’s IP Address that passes network traffic between different IP networks.
Router Preference

Enter a numerical router preference.

We have more than one network defined in our deployment (admin, public, and os_sdn ). And, each network requires a set of IP address configurations, including a router. When a node gets more than three networks and three routers defined, the packets going out of the node outside of these networks need to know which router IP address has to route the traffic. The Router Preference determines the router order.
Create Network

Click to save the settings.
