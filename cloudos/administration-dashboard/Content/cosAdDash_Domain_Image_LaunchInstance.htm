How to Launch an Image

The cloud admin can launch images on the Cloud > Images window. The Cloud OS uploads the image to your image service.

The image must be available on an HTTP URL. Compressed image binaries, such as .zip, and .tar.gz, are supported.

You can only launch images that are formatted as AMI, RAW, and QCOW2. Launching an image provisions it to the cloud.

 

To launch an image:

  1. Select Images to open the Images view.

  1. Find the image to launch and then click Launch to open the Launch Instance dialog box. The dialog opens to the Details tab. Some fields are automatically populated from the image.

  2. Enter or edit the Details tab information as needed:

    1. Instance Source — Specify whether you are using an image or snapshot.

    2. Image — Select from the list of images that have been uploaded to the system.

    3. Instance Name — The name of the host you are creating (Required).

      Note: At this point, you can go ahead and launch the instance, using the defaults for the remaining options.

    4. Flavor — Indicates the size of the node. When you select a flavor here, the Flavor Details table on the right changes to show the node properties and values associated with your size selection.

    5. Instance Count — Can be more than 1.

  3. Use the Access & Security tab to control access to your instance via keypairs and security groups.

    1. Keypair — Keypairs are SSH security keys that are 512 bits long, provide authentication, and allow you to exchange traffic securely within a VM. These credentials are injected into images when they are launched.

      Creating a new keypair registers the public key and downloads the private key (a .pem file), allowing you to log into the VM. The public key gets installed on the VM; the private key is used to log in.

      To add a keypair, either:

      • Select a keypair from the list, or

      • Click (+) to import a keypair. Enter Keypair Name and Public Key

  4. Use the Networking tab to select one or more networks. You can click the plus (+) symbol or drag and drop to make your selections. Use the minus (-) symbol to remove networks.

  5. Use the Volume Options tab to select boot options for an instance that launches with attached storage.

  6. Use the Post Creation tab to customize the instance by adding a script to run after launch. The Customization Script field is analogous to "User Data" in other systems.

  7. Click Launch. If successful, the image is added to the Images list with an Active status.
