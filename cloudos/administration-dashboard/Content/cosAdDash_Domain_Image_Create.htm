How to Create an Image

Cloud administrators can create images on the Domain > Images window.

The image must be available on an HTTP URL. Compressed image binaries, such as .zip, and .tar.gz, are supported.

To create an image:

  1. Click Cloud > Images to open the Images window.

  2. Click Create Image to open the Create an Image dialog box. Enter:

  3. Name.

  4. Image Location that meets the following requirements:

    1. A valid and direct HTTP URL to the image binary. URLs that redirect or serve error pages will result in unusable images.

      Note: If you do not have rights to the image binary url, you receive an error when trying to create the image

    2. The image must be accessible to the Image Service.

    3. The image must be in a supported format.

      Note: If you upload an image with a format that is not supported, you will receive an Error 404, Not Found error when you attempt to create the image.

  5. Select an Image File.

    Select a Format from the drop-down list. The image must be in one of the formats listed.

  6. Enter the Minimum Disk space and Minimum Ram.

  7. Keep Public selected if you want the image visible to all users.

  8. Select Protected to mark the image as read-only. The image cannot be edited or deleted.

  9. Click Create Image. If successful, the image displays with an Active status. If creation is not successful within a specified amount of time (set by your system administrator; typically no more than five minutes), the Status of the image in the list displays Upload Failed.

Note: Only AMI, RAW, and QCOW2 images are launchable.
