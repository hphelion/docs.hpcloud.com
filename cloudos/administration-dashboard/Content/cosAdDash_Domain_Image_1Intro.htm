Image Repositories

 

Cloud administrators can create image repositories on the Cloud > Images window and make the images public. Users can access those images on Project Infrastructure > Images and use them to launch a VM.

When you launch a VM, the launch is based on an image. Images shows all non-snapshot images with the following formats:

AMI (Amazon Machine Image) — Not valid unless it is associated with a kernel image (AKI) and a RAMdisk Image (ARI).

ISO — Optical Disk Image

QCOW2 — QEMU Emulator

RAW

VDI — Virtual Disk Image

VHD — Virtual Hard Disk Image

VMDK — Virtual Machine Disk Image

Images are stored at the Domain level with a single Glance service and are unaffected by region selection. Only Public images are visible to all users.

 

To display a list of domain images:

  1. In the left navigation frame, click Images.
  2. To filter this list, click () and enter the first several characters of the Name or consecutive characters within the Name. Then click Filter All. To reset, click Filter All again. (<Enter> works the same as Filter All.)

 

To display image details:

  • Click the linked Image Name. The Image Overview window opens with read-only image information and specs.

An AMI image displays additional custom properties for the Kernel ID and RAMdisk ID, which are cross links to those associated image details.
