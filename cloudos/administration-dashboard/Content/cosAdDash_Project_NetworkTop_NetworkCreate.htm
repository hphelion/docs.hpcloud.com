How to Create a Network Topology

A network is an isolated, virtual, layer-2, broadcast domain that is typically reserved for the tenant who created it, unless the network is configured to be shared. Tenants can create multiple networks until they reach the thresholds specified by per-tenant.

 

To create a project topology network:

  1. On the Project tab, select a Current Project.

  2. Click Infrastructure > Network Topology to see the layout diagram.

  3. Click Create Network to open the Create Network dialog box.

  4. Keep Admin State selected to create a network that is active and operational.

  5. In the Create Network window, enter a Network Name and then click Create.

  6. If you do not want to create a subnet that is associated with this topology network, in the Subnet tab, uncheck Create Subnet.

  7. If you want to create a subnet that is associated with this network, keep Create Subnet checked.

    1. On the Subnet tab, enter a Subnet Name and Network Address.

    2. From the drop-down list, select an IP Version.

    3. (Optional) Enter a Gateway IP. This is the IP address that servers on the subnet will use if the address they want to send to is not on the subnet. It is a gateway to other networks. In a common use case, the gateway address would be that of a router; however, it could also be of another server that is providing routing services.

    4. Keep Disable Gateway unchecked. Keep this unchecked if you prefer that any messages sent outside the subnet should fail rather than be forwarded to their destination. It becomes a private subnet that packets cannot be sent out of, except through other devices on the network.

  8. Click Create.

  9. Verify that the topology network you just created is included in the Networks list.
