How to Delete, Disable, or Enable a User

Cloud administrators can delete, disable, and enable users.

Note: You cannot delete, disable, or enable the following users: Admin, Arch, and control users such as Trash or Nova. Delete, Disable, and Enable do not display for those users.

 

Deleting a user marks the user for deletion by adding a ~ (tilde) prefix to the name. The user will be permanently removed from this list when the clean_users script runs. (The script is scheduled by a system administrator).

 

To delete a user:

  1. Click Domain > Users to open the Users window.
  2. Find the user you want to delete. Click More > Delete User.
  3. A confirmation dialog box opens, warning you that the action cannot be undone. To confirm, click Delete User.
  4. The Users list redisplays and the user is marked for deletion. The deleted user will be permanently removed from the list when the clean_users script runs.

 

A deleted user is not removed from the Administration Dashboard until the clean_users script runs. You can restore a deleted user before the script runs.

To restore a deleted user:

  1. Click Domain > Users to open the Users window.

  2. Find the user you want to restore and click Restore User.

  3. A confirmation dialog box opens. To confirm, click Restore.

  4. The Users list redisplays and the user moves back into the list alphabetically.

Note: After you restore a user, the user is no longer associated with any project.

 

You can disable a user, so the user remains in the Administration Dashboard, but cannot log in.

To disable a user:

  1. Click Domain > Users to open the Users window.

  2. Find the user you want to disable. Click More > Disable User.

  3. The user remains on the list but the Enabled field changes to False.

     

You can enable a disabled user. To enable a user:

  1. Click Domain > Users to open the Users window.

  2. Find the user you want to enable. Click More > Enable User.

  3. The user remains on the list and the Enabled field changes to True.
