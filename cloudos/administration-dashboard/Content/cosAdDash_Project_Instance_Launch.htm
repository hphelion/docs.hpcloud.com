How to Launch an Instance

Caution: When you set up an instance, carefully make your selections. After the instance is launched, the only editable property is its Name.

 

To launch an instance:

  1. On the Project tab, select a Current Project.

  2. Click Infrastructure > Instances to open the Instances window.

  3. Click Launch Instance to open the Launch Instance window.

  4. The window opens at the Details tab. Enter the following information:

    1. Instance Source — Whether you are launching an image (default) or snapshot.

    2. Image — Depending on your selection in the Instance Source field, you are required to select an AMI, RAW, and QCOW2 formatted image or a specific snapshot. (ARI and AKI types cannot be launched.)

    3. Instance Name — The name of the host you are creating (Required).

    4. Flavor — Indicates the size of the node. When you select a flavor here, the Flavor Details table on the right changes to indicate the node properties and values associated with your size selection.

      1. Name — This is tiny, small, medium, large, or xlarge, or the name of a custom flavor that has been created
      2. CPUs — The number of virtual CPUs.
      3. Root Disk — Assigned by the image.
      4. Ephemeral Disk — A second storage disk available to the VM.
      5. Total Disk— This size is defined in Region > Flavors by the Domain Administrator. It is known as the Root Disk.
      6. RAM — Processor memory.
    5. Instance Count — Can be more than 1.

  5. Use the Access & Security tab to control access to your instance via keypairs and security groups.

    1. Keypair — Keypairs are ssh credentials that are injected into images when they are launched.

      Select a keypair from the list or click (+) to import a keypair. Creating a new keypair registers the public key and downloads the private key (a .pem file).

    2. Security Groups — Security groups are one way to provide a measure of network isolation between different VMs, as they limit the ports into which network traffic can enter the group. You can select multiple groups.

  6. Use the Networking tab to select one or more networks. You can use the plus (+) symbol or drag-and-drop to select networks. Use the minus (-) sign to remove networks.

  7. Use the Volume Options tab to select boot options for an instance that is launching with attached storage.

  8. Use the Post Creation to customize the instance. This tab is analogous to "User Data" in other systems. The tab allows you to customize the instance by adding a script to run after instance launch.

  9. Click Launch. If the launch is successful, the instance is added to the Instances list with a Power State of Running.
