How to Launch an Instance

You can launch an instance into a project, if you are a member of that project. Even a cloud administrator must be a member of a project to launch an instance into that project.

Make your selections carefully when setting up an Instance, as the only editable property after it is launched is the Name.

 

To launch a region instance:

  1. On the Region tab, select a Current Compute Region.

  2. Click Instances > Launch Instance.

  3. (Optional) Select the Project and User for whom the instance should be created.

  4. Select the Details tab to specify the details for launching an instance.

    1. Instance Source - an Image (default) or Snapshot

    2. Image - Depending on your selection in Instance Source, you are required to select an AMI, RAW, or QCOW2 formatted image to create the new VM or a specific snapshot.

    3. Server Name - Name the host you are creating (required) NOTE: The instance can be launched at this point, if desired, with system defaults for remaining options.

    4. Flavor - Indicates the size of the node. When you select a flavor here, the Flavor Details table on the left changes to indicate the node properties and values associated with your size selection.

      1. Name - tiny to xlarge or the name of a custom flavor that has been created

      2. Number of virtual CPUs

      3. Root Disk - The one that gets assigned by the image

      4. Ephemeral Disk - A second storage disk available to the VM

      5. RAM - Processor memory

      6. Instance Count - Can be more than 1

  1. Select the Access & Security tab to control access to your instance via keypairs and security groups.

    1. Select a keypair - Keypairs are ssh credentials which are injected into images when they are launched. Click (+) to import a keypair. Creating a new keypair here registers the public key and downloads the private key (a .pem file).

    2. Security Groups - Security groups are one way to provide a measure of network isolation between different VMs, as they limit the ports into which network traffic can enter the group. You can select multiple groups.

  2. Select the Volume Options tab to select boot options for an instance launching with attached storage.

  3. Select the Post Creation tab to customize the instance by adding a script to run after launch of the instance. (The Customization Script field is analogous to "User Data" in other systems.)

  4. Click Launch. The Instance is added to the Instances list with a Power State of Running.
