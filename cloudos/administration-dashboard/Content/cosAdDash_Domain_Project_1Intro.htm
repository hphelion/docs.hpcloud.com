Projects

There is only one set of projects. Under Cloud Projects, all projects are visible. Under Domain Projects, only the projects visible to that domain are listed.

Both a Cloud Administrator and a Domain Administrator can assign users to projects through the use of roles as long as the project is in the same domain as the Domain Administrator.

 

Projects are groupings that allow designated users to view the same set of content. Projects protect topology templates, infrastructure designs, and realized resources (such as VMs, volumes, and others) from being visible to all users.

Cloud Administrators and Domain Administrators are responsible for creating projects and securing them by role assignments.

See Administration Dashboard Roles.

To display a list of projects:

  1. Select Projects to open the Projects view. Projects are listed alphabetically by Name.

  2. To filter this list, click () and enter the first several characters of the Name or consecutive characters within the Name. Then click Filter All. To reset, click Filter All again. (<Enter> works the same as Filter All.)

 

To display the details of a project:

  1. Click a Project Name link.
  2. The Project Detail Overview view opens with read-only project information and a list of users and roles associated to the project.
  3. (Optional) Click a User link to open the Users Overview window.
  4. (Optional) Click a Role link to open the Roles Overview window.
