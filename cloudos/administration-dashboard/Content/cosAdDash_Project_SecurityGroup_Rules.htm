How to Add or Delete a Security Group Rule

Within a security group, you can add, edit, and delete rules to specify communications to other ports.

 

To create a security group rules:

  1. On the Project tab, select a Current Project.

  2. Click Access and Security > Access and Security > Security Groups tab.

  3. Select the group to edit, and click Edit Rules.

  4. In the Security Group Rules view, click Add Rule.

  5. In the Add Rule window, enter the following information:

    1. IP Protocol — Select from the drop-down list.

    2. Open — Select from the drop-down list.

    3. Port Range:

      1. From Port — The source port, such as 80.
      2. To Port — The port where VMs inside the security group would receive, such as 80.

    4. Source: The IP addresses from which the group can receive networking traffic

      1. CIDR — The format for specifying a range (a block) of IP addresses that includes how many to create.
      2. Security Group — The source group will allow any other instance in this security group access to any other instance.
  6. Click Add.
  7. Wait for the success message.
  8. Verify that the new rule is included in the Security Group Rules list.

 

To delete security group rules:

  1. In the Security Group Rules list, select the rule you want to delete.

  2. Click Delete Rule.

  3. In the Confirm Delete Rule dialog, verify the name of the rule and then click Delete Rule.

  4. Wait for the success message.

  5. Verify that the new rule is excluded from in the Security Group Rules list.
