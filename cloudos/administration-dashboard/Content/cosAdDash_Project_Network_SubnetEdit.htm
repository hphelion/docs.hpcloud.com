How to Edit a Subnet

The Region > Networks window provides a convenient place for the admin user and cloud administrators to see all the subnets in all the regions, so administrators can decide which subnets to edit.

In many cases, especially if the subnet is associated with an external network, subnet details change after it is created. As a result of these changes, you might need to update information. For example:

  • If you entered an incorrect subnet gateway address during subnet creation, or if the subnet gateway address changes, you can correct the subnet gateway address. You cannot change the mask of a subnet.
  • You can change or add DNS servers, and modify routing information.
Note: Only the Admin user can edit a subnet on an external network.

 

To edit a subnet:

  1. Click Region > Networks to open the Networks window.

  2. Click the Network Name where you want to edit a subnet. The Administration Dashboard opens the Network Overview window in the associated project. You are now working at the project level.

  3. Find the subnet you want to edit, and click Edit Subnet to open the Update Subnet dialog box where:

    1. On the Subnet tab of the dialog box, you can change the Subnet Name, Gateway IP, or Disable Gateway settings. You cannot edit the Network Address.

    2. On the Subnet Details tab of the dialog box, you can change the Enable DHCP setting, DNS Name Servers, or Host Routes settings.

    3. Click Update.

  4. The Network Overview window opens, and you see the edits.
