How to Create a Network

You can create a network that is assigned to your current project.

Note: You cannot create an external network on the Project tab. You must have access to the Region tab to create a shared or external network. See How to Create an External Network.

 

To create a project network:

  1. On the Project tab, select a Current Project.

  2. Click Infrastructure > Networks to open the Networks window.

  3. Click Create Network to open the Create Network dialog box

  4. On the Network tab of the dialog box:

    1. Enter a Network Name for your network.

    2. Make sure Admin State is checked. This is the administrative state of the network. If you select Admin State, the network forwards packets. If you do not select Admin State, the network does not forward packets.

  5. On the Subnet tab of the dialog box:

    1. To create a network with no associated subnet, unselect Create Subnet and click Create. You are done.

    2. If you want to create a network with an associated subnet, keep Create Subnet selected and continue through the following steps:

      1. Enter a Subnet Name and Network Address.

      2. Select an IP Version.

      3. (Optional) Enter a Gateway IP. This is the IP address that servers on the subnet will use if the address they want to send to is not on the subnet. It is a gateway to other networks. In a common use case, the gateway address would be that of a router; however, it could also be of another server that is providing routing services.

      4. Select Disable Gateway if you want messages that are sent outside the subnet to fail: the messages will not be forwarded to their destination.

        When you disable the gateway, the subnet becomes private. Packets cannot leave the subnet, except through other device on the network.

        1. Leave Enable DHCP selected.

        2. Enter an Allocation Pools IP address list for the subnet, using one line in the entry box for each address.

        3. Enter a DNS Name Servers IP address list for the subnet, using one line in the entry box for each address.

        4. Enter a Host Routes IP address list to announce additional routes to the hosts. Use one line in the entry box for each address.

  6. On the Subnet Detail tab of the dialog box:

    1. Leave Enable DHCP selected.

    2. Enter an Allocation Pools IP address list for the subnet, using one line in the entry box for each address.

    3. Enter a DNS Name Servers IP address list for the subnet, using one line in the entry box for each address.

    4. Enter a Host Routes IP address list to announce additional routes to the hosts. Use one line in the entry box for each address.

  7. Click Create.

  8. Verify that the network you just created is included in the Networks list.
